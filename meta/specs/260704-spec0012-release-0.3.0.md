# spec0012 — corte da release 0.3.0 (bump de versão + fechamento do CHANGELOG)

**Tipo:** release/docs · **Alvos:** `flatdrop/__init__.py`, `meta/CHANGELOG.md`
**Autor:** chat · **Aplicador:** Claude Code
**Depende de:** spec0011 aplicada (o `_TREE.md` é o último item de produto da 0.3.0).

## Objetivo

Cortar a **0.3.0**. Todo o corpo acumulado em `[Não lançado]` (specs 0001–0011) já
está aplicado, commitado e com suíte verde (35 testes). Esta spec: (1) bumpa a
versão no `__init__.py`; (2) acrescenta ao CHANGELOG a entrada do `_TREE.md` que
ainda faltava; (3) data a seção como `[0.3.0]` e abre nova `[Não lançado]` limpa.

Sem mudança de comportamento de código — é corte de release. Nenhum teste novo.

## Contexto / verificação

- `flatdrop/__init__.py` hoje traz `__version__ = "0.2.0"`.
- O CHANGELOG tem `[Não lançado]` descrevendo 0001–0010, mas **não** cita o
  `_TREE.md` (spec0011, aplicada depois). Precisa entrar antes de datar.
- A 0.3.0 é MINOR (adições retrocompatíveis: CLI, multi-fonte, `.flatdropignore`,
  tipos, gerador de `.bat`, `_TREE.md`; mais FIX-003/FIX-004). SemVer confere.
